# 🧠 Сбер.Старт — AI-аналитика бизнес-идей

> **MVP платформы для снижения неопределённости и страха старта бизнеса**

Сбер.Старт — это интеллектуальный аналитический модуль, который помогает начинающим предпринимателям оценить жизнеспособность своей бизнес-идеи до открытия бизнеса.

Вместо субъективных советов сервис сопоставляет идею пользователя с тысячами похожих бизнес-сценариев, анализирует статистику их развития и формирует объективный аналитический отчет с помощью искусственного интеллекта.

---

# 🎯 Главная гипотеза MVP

> **ИИ способен снижать неопределённость и страх начала бизнеса за счёт сопоставления идеи пользователя с реальными паттернами похожих бизнесов.**

MVP не принимает решения за пользователя.

Его задача — предоставить объективную data-driven аналитику, основанную на опыте аналогичных предприятий.

---

# 🚀 Что умеет MVP

После ввода бизнес-идеи система автоматически проходит несколько этапов анализа.

### 1. Ввод идеи

Пользователь описывает свою идею свободным текстом.

Например:

> Хочу открыть небольшую кофейню формата to-go рядом с университетом.

---

### 2. Построение цифрового профиля

На основе текста строится цифровой профиль бизнес-идеи.

Например:

- ниша;
- целевая аудитория;
- масштаб;
- формат бизнеса;
- предполагаемые инвестиции.

В текущей версии MVP этот этап может выполняться как напрямую через embedding-модель, так и с помощью LLM.

---

### 3. Поиск аналогичных бизнесов

Идея преобразуется в векторное представление.

Далее выполняется поиск наиболее похожих бизнес-сценариев в FAISS.

В качестве базы используются синтетические обезличенные кейсы, которые легко могут быть заменены реальными данными.

---

### 4. Анализ статистики

По найденным аналогам рассчитываются:

- средняя вероятность успешного развития;
- наиболее распространённые причины закрытия;
- основные факторы успеха;
- общая статистика по выбранной нише.

---

### 5. Генерация аналитического отчёта

Полученная статистика передается в локальную LLM (Ollama), которая формирует:

- ключевые риски;
- рекомендации;
- краткое резюме;
- аналитическое заключение.

Важно отметить, что LLM не придумывает информацию самостоятельно.

Она лишь интерпретирует агрегированные данные, найденные системой.

# 🚀 Быстрый старт

## 1. Клонирование репозитория

```bash
git clone <repository_url>
cd sber_start
```

---

## 2. Создание виртуального окружения

### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

### Linux / macOS

```bash
python -m venv venv
source venv/bin/activate
```

---

## 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### Пример requirements.txt

```text
fastapi==0.104.1
uvicorn[standard]==0.24.0
sentence-transformers==2.2.2
faiss-cpu==1.7.4
requests==2.31.0
pandas==2.1.3
numpy==1.24.3
python-dotenv==1.0.0
```

---

## 4. Установка Ollama

Установите Ollama.

После установки загрузите модель:

```bash
ollama pull qwen2.5:7b
```

Запустите сервер:

```bash
ollama serve
```

Если используется другая модель, измените её название в:

```
app/services/llm_service.py
```

---

## 5. Создание базы кейсов и FAISS-индекса

Перейдите в backend:

```bash
cd backend
```

Запустите:

```bash
python -m app.scripts.init_db_and_faiss
```

Будут автоматически созданы:

```
backend/data/
    mock_cases.json
    faiss_index.bin
```

При необходимости синтетические данные можно заменить реальными.

---

## 6. Запуск API

```bash
uvicorn app.main:app --reload
```

или

```bash
python -m app.main
```

---

## 7. Запуск интерфейса

После запуска откройте браузер:

```
http://127.0.0.1:8000
```

Введите описание идеи и нажмите кнопку анализа.

# ⚙ Настройка проекта

## Использование другой LLM

В файле

```
app/services/llm_service.py
```

измените

```python
model_name="qwen2.5:7b"
```

на необходимую модель.

---

## Расширение базы данных

Добавьте записи в

```
mock_cases.json
```

Каждая запись должна содержать:

```json
{
  "id": 1,
  "niche": "кофейня",
  "description": "...",
  "success_rate_percent": 65,
  "common_risk": "...",
  "top_risks": [],
  "success_factors": [],
  "survival_12_months": true
}
```

После изменения данных необходимо снова выполнить:

```bash
python -m app.scripts.init_db_and_faiss
```

---

## Изменение количества похожих кейсов

В

```
main.py
```

можно изменить

```python
similar_cases = vector_db.search_similar(
    idea_text,
    k=30
)
```

Чем больше значение `k`, тем более устойчивой становится статистика.

# 🛠 Возможные ошибки

| Ошибка | Причина | Решение |
|---------|----------|----------|
| ModuleNotFoundError | Запуск не из backend | Перейдите в backend и используйте `python -m app.main` |
| Ошибка подключения к Ollama | Ollama не запущен | Выполните `ollama serve` |
| FileNotFoundError | Не создан FAISS | Запустите `init_db_and_faiss.py` |
| Ошибка JSON | LLM вернула невалидный JSON | Используйте `format="json"` |
| Медленный запуск | Загружается embedding-модель | Происходит только один раз |
| Нет результатов поиска | Недостаточно данных | Добавьте кейсы или увеличьте `k` |

# 💰 Стоимость использования

При использовании Ollama все вычисления выполняются локально.

- отсутствует стоимость запросов;
- отсутствуют ограничения API;
- все данные остаются на компьютере пользователя.

# 📝 Для жюри

Данный MVP демонстрирует не просто использование LLM, а полноценный инженерный подход к построению интеллектуальной системы поддержки принятия решений.

Основные особенности решения:

- использование векторной базы данных FAISS;
- поиск по семантическому сходству бизнес-идей;
- применение современных embedding-моделей;
- статистическая агрегация найденных кейсов;
- генерация аналитического отчёта исключительно на основе найденных данных;
- модульная архитектура, позволяющая заменить синтетические данные реальными без изменения логики системы.

Таким образом, пользователь получает не субъективный совет нейросети, а объективную аналитическую оценку, основанную на опыте аналогичных бизнесов.
